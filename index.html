HTML

HTML (Hypertext Markup Language) gives a web application the content structure. It is essentially like building the frame to a house. It is made up of elements that work together to provide content and the basic framwork of all pages on the web.

At the top of the webpage, we see the !DOCTYPE html declaration. This declaration tells the browser what document type it should expect when it is open. The DOCTYPE is not case-senstiive because it is a declaration. Declarations are not case-sensitive; HTML elements are case-sensitive.

Elements

An element is a type of content on a webpage. Common elements include:

  • Text elements
  • Image elements
  • Header elements
  • List elements (like this one!)

In addition to elements that can be seen on the page, elements are also used to divide the content on a page to describe the content to browsers and other developers.

Element types that are used to divide and describe:

  • Meta-related elements
  • Content division elements
  • Semantic elements

Head element

The head element contains information about the webpage, or metadata, that helps the browser render the page correctly. Elements within the head are not visible to the user.

Metadata is data about our webpage that the browser uses to effectively serve our webpage to users. Metadata is typically not seen by users, but it contains important information about accessibility, search engines, and performance.

  • The style element defines the CSS styles applied to the page.
  • The base element defines the base URL for a webpage.
  • The link element connects an external resource to the HTML document.
  • The meta element defines character set, description, keywords, author, and viewport.
  • The script element embeds client-side scripts in the HTML document.

In our example of the Panda Fan site, the meta element defines:

  • The charset, or character set, as UTF-8, which allows us to use most characters and symbols on the page.
  • The http-equiv element sets an HTTP header for content.
  • The viewport element sets the default view for our webpage.

Body element

The body element represents the visible content shown to the user.

Section elements

Section elements are semantic elements that define a section in a document that contains thematic content.

HTML uses the id attribute as a way to identify elements and select them.

CSS

CSS, Cascading Style Sheets, is a programming language used to define how webpage content is presented to others. CSS defines the colors, aesthetics, visual layout, and organization.

When a developer wants to style an element directly in the HTML file, they use inline CSS. They are useful for quick and premanent changes but are less flexible than external an internal style sheets.

It is also possible to embed a stylesheet directly in the HTML file using the style element that is nested in the head. The rules only apply to the page, and you can configure CSS classes and IDs.

The most common way to style a webpage is by using an external CSS style sheet. When using an external style sheet, link the file using a link element placed in the head element.

CSS Rules

  • A selector defines the element or attributes to which the rules, or declaratations apply.
  • Class attributes allow us to share a CSS rule to any element by assigning the rule to a class attribute with a class selector.
  • A period preceding the class designates it as a class selector.
  • Declarations contain two important components: the CSS property we want to apply and the value of the property.

CSS Box Model

Think of every HTML element as a box. Each box consists of margins outside the element, and the element's borders, padding, and content.

  • A margin property indicates how much space we want around the outside of an element.
  • The padding property adds space around the content inside an element. When you increase padding, you are making the element bigger.

DRY Code

Do Not Repeat Yourself (DRY) code helps keep code as succint as possible. Developers often combine two or more selectors into a single CSS rule with a comma.

CSS Properties & Values

  • A display property assigned with a block values tells an element to start on a new line.
  • The rel attribute in a link element specifies the relationship between the current document and the linked document source.
  • The box-shadow CSS property adds shadow effects around an element's frame.
    • Multiple effects can be separated by commas
    • A box shadow is descibed by X and Y offsets relative to the emene,t blue and spread radius, and color.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -A: adds code to the working branch to the staging area. The staging area informs Git that the changes will be added in the next commit.
  • git commit: entered with a commit message (-m "commit message") that describes what the commit contains.
  • git pull origin < branch name >: receives a branch's modifications to the local environment, with the origin indicating the source of the pull will be in the GitHub repo. The fourth keyword "main" indicates the branch.
  • git push origin feature/starter-code: Pushes changes to the remote branch. Push pushes the changes we have locally to our remote GitHub branch.

JavaScript

JavaScript (JS) is a high-level dynamic programming language that allows users to interact with webiste. Through JavaScript, you can power webistes with effects, interactivity, data communication, and more.

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • Arrays can be used to store data types, and a for loop can be used to operate on each element in an array.
  • A function is a set of instructions that tells the computer how to perform a certain task.

A for loop uses the predictable pattern of indices to perform a task on all the items in an array by allowing a single code block to be executed over and over.

A for loop contains the following three important statements:

  1. The first statement determines the starting point for our loop.
  2. The second statement is the condition.
  3. The final statement is what allows the array to iterate over each item.
